Dynamic elements

Provided by javascript, dynamic elements allow an HTML page to interact with the user, itself, other pages, and databases outside of it and or its home network. I will be using these elements to update the page as the game, which this site will be, progresses. DOM-wise, for this web-app game, there will be only one html file, the index. The contents of this page will update every time the player makes a choice in the game as if the user has changed pages.

Wireframes

Main Menu [page 1] Frame

Logo:
  Probably a "B" for brave in "Brave The Darkness";

Title:
  Brave The Darkness;

Description:
  You will be placed in a predicament that requries one to face the abyss and find a way out of the wilderness. Will you brave the darkness?;

Navigation Links:
  Home Page;

Buttons:
  Brave, Flee;

Colors:
  shades of green, slightly muted;

Step [page 2] Frame

Logo:
  Probably a "B" for brave in "Brave The Darkness";

Navigation Links:
  Home Page, Main Menu, Map (optional), Credits;

Step Image:
  Typically a landspace of the setting in which the user is currently in within the game.

Step Name:
  An artisitic name of the step the user is in. For example, the setting of the first step is a clearing within the trees. So, the name of this step could be one of the following: "The Center Place", "The Start of Your Journey", or simply "The Clearing."

Step Narration:
  This game follows an underlying story/narrative. Therefore, the steps are narrated along the way. One only begins to understand the story once they are able to finish the game successfully. The narration describes the current setting in which the user is in and how they may proceed with the game.

Step Decisions:
  There are anywhere between 2 and 4 decisions for each step. They will be placed in proximity to the step narration. Currently, it is planned to by below the narration.
